Visualization#

Visualization related functionality are provided in sub package :module: navground.sim.ui and not imported in the top-level package to minimize dependencies.

Synchronize HTML clients with a simulation#

module:

navground.sim.ui

class WebUI#

Displays the world on HTML client views. The state is synchronized through websockets.

Variables:

decorate (typing.Mapping[str, str]) – A function to decorate entities, called at each update

After creating an instance, you need to call prepare() to finalize the initialization of the websocket server, else the HTML clients will not be able to connect.

>>> ui = WebUI()
>>> await ui.prepare()
>>> # initialize the views with the current state of the world
>>> world = ...
>>> await ui.init(world)
>>> # update the world, for example with
>>> world.update(0.1)
>>> # synchronize the views
>>> await ui.update(world)
>>> # set entity attributes, for example to change the color of the first agent
>>> await ui.set(world.agents[0], style="fill:red")
>>> # or analogously
>>> await ui.set_color(world.agents[0], "red")

In general, users will not use this class directly but delegate updating the view to navground.sim.RealTimeSimulation or navground.sim.RealTimeReplay, like the following:

>>> ui = WebUI(host='127.0.0.1')
>>> await ui.prepare()
>>> sim = RealTimeSimulation(..., web_ui=ui)
>>> await sim.run()
